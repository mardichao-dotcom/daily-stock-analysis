## 變更摘要

| 條目 | v2.1 | v2.2 |
|---|---|---|
| **區域類觸發** | 收盤在區內(low ≤ close ≤ high) | **K 棒碰到區域**(K_low ≤ high AND K_high ≥ low) |
| **站穩 Day 2 視窗** | 最多 3 個交易日內可達成 | **嚴格隔天**(Day 1 → Day 2 必須是相鄰交易日) |
| **站穩取消** | 連續 2 天 close<p OR 一天 open<p AND close<p(僅向下) | 連續 2 天 close<p OR 一天「離開給定價」(**雙向** — 全在上方也算) |
| **跌破** | state(prev STANDING/MAINTAINING 才可能跌破) | **event**(Day 1 + Day 2 自成,不需先 standing) |
| **C 級分組** | 列所有 active 站穩 / 跌破 tags | **只列「當天新成立」**;持續站穩中僅在 K 線上顯示 |

## §1-D. 給定價格類觸發條件(修訂)

### 線類(維持 v2.1)
- K 棒「碰到」線:`K_low ≤ given_price ≤ K_high`
- 站穩 / 跌破見 §3-A

### 區域類(新)
| Category | v2.1 觸發 | v2.2 觸發 |
|---|---|---|
| 訂單塊 | close ∈ [low, high] | **K_low ≤ high AND K_high ≥ low** |
| POC | 同上 | 同新 |
| FVG | 同上 | 同新 |
| 跳空缺口 | 同上 | 同新 |

**邏輯**:K 棒範圍跟區域範圍有交集即觸發(碰到就算)。不要求收盤在區內。

## §3-A. 站穩 / 跌破定義(修訂)

### 站穩成立(兩天確認)

**Day 1(碰到當天)**:
```
1. K 棒碰到給定價   K_low ≤ given_price ≤ K_high
2. 收盤 ≥ given_price
→ 🟢 第一個綠圈(內部 state: TRIGGERED,trigger_date=Day1)
```

**Day 2(隔天)**:
```
3. open  ≥ given_price
4. close ≥ given_price
→ 🟢 第二個綠圈
→ 站穩成立(STANDING,standing_date=Day2)
```

如果 Day 2 沒符合 → 該觸發作廢,等下次重新碰到才再起算。
(不允許 Day 3+ 補位,跟 v2.1 不同)

### 站穩取消(只在 active 狀態才檢查)

| 條件 | 描述 |
|---|---|
| (a) 連續 2 天收盤 < given_price | 跟 v2.1 相同 |
| (b) 一天「open + close 都離開 given_price」(**雙向**) | **v2.2 新增** |

「離開 given_price」精確定義:
```
leave_up   = (open > given + ε) AND (close > given + ε)   # 全在上方
leave_down = (open < given − ε) AND (close < given − ε)   # 全在下方(跟 v2.1 down_day 相同)
leave      = leave_up OR leave_down
ε = 0.01(避免浮點)
```

→ 滿足取消條件:`active = false`,寫 `breached_date = today`(欄位重用,
語意改為「失效日」)

### 跌破成立(event,兩天確認)

**Day 1**:
```
1. K_low ≤ given_price ≤ K_high
2. close ≤ given_price
```

**Day 2**:
```
3. open  ≤ given_price
4. close ≤ given_price
→ 跌破 event(一次性,Day 2 當天觸發 🔴 標籤 + chart event marker)
```

**重點**:跌破跟「之前有沒有站穩過」**無關**。
即使是處女關鍵價,如果 K 棒先碰到、Day 2 又被壓下,就觸發跌破 event。

不維護「跌破 state」,純 event。

## §3-B. 加分時機(無變更)

```
✅ 站穩成立當天(Day 2) +N(只算 1 次)
✅ 跌破後重新站上 → 可以再 +N
✅ 區域類:K 棒碰到當天 +N(注意:v2.2 起區域不需要兩天確認,碰到即算)
   (與線類站穩 +N 的時機差異:線類強調「站穩確認」,
    區域類強調「進入區域」)
✅ 跌破不扣分(純加分制)
✅ 每日重算當日分數,過了就清零
```

### v2.2 區域 vs 線類 加分時機對照

| 類型 | 觸發時機 | 加分時機 |
|---|---|---|
| 線(站穩) | Day 1 碰到 → Day 2 確認 | **Day 2 確認當天 +N** |
| 線(跌破) | Day 1 碰到 → Day 2 確認 | 不加分,Day 2 發 🔴 標籤 |
| 區域 | K 棒交集當天 | **當天 +N** |

## §4. 標籤體系(C 級分組新增「今天新成立」限制)

| 標籤 | v2.1 列入 C 級條件 | v2.2 列入 C 級條件 |
|---|---|---|
| 🟢 站穩 X | active(STANDING / MAINTAINING) | **只列 standing_date == today**(當天 Day 2 確認) |
| 🔴 跌破 X | event(當天觸發) | event 當天(同 v2.1) |
| ⚡ MACD 動能轉多 | 當天 | 當天 |
| ⚡ MACD 動能轉空 | 當天 | 當天 |
| ⭐ 個股輪動 | 當天 | 當天 |
| ⛔ ETF 減碼 | 當天 | 當天 |

**持續站穩中**(active 但 standing_date < today):
- 不列在 C 級分組(避免「同檔多年累積站穩」灌水)
- K 線圖上仍顯示「🟢 站穩中」標籤(歷史脈絡)
- 個股卡片內 tags 仍有(完整資訊保留)
